classHidden string ="hidden" Name of a CSS class which should be added to the toggle element (see property toggle) when hiding the corresponding section and removed when showing the section.
easing string <optional>
 A string indicating which easing function to use for the show or hide transition. See jQuery.animate(). If undefined (default), jQuery.animate()'s default easing function will be used.
horizontalAnimation boolean =false: Default tries vertical easing when showing or hiding a section, set to true in order to change to horizontal easing.
duration int =200 Default duration for animations (scrolling, or easing when expanding or collapsing elements).
scroll boolean =false. When false (default), any call to showSection() or showOrHideSection() (for showing) will simply show the target section regardless of whether it is in the viewport or not, i.e. not ensuring that the user will see the newly showing section. Set this to true to ensure that the newly shown section will be scrolled into viewport: If the section is below viewport, the page will scroll down until the section is in view, if the section's top is above viewport (the section might be partially showing or not at all, but the start of the section is not in view), the page will scroll up to show the beginning of the newly shown section.
scrollTriggerTolerance int <optional>
 =50 Tolerance for the decisions if/when to trigger a scroll. Only evaluated, of course, if option scroll is true. See tolerance parameter of jQuery.fn.isTopBelowViewport resp. jQuery.fn.isTopAboveViewport.
scrollTolerance int <optional>
 Tolerance for the scroll operation (see option scroll}). If left undefined (default), the default tolerance of the called scroll function (jQuery.fn.scrollDownIntoView resp. jQuery.fn.viewportAlignTop) will be used.
toggle string = "#${id}_h" String pattern for a jQuery selector string used to find the section heading element (usually containing a toggle control) whose class list is to be changed in order to reflect the visibility state of the section. For example: If you have a link or button and have bound a function to it's click event in order to call $("#mySection").showOrHide(), i.e. a link or button which is used to toggle the visibility of a section with the example ID "mySection", then you may define this link or button to be the toggle control. When the section in invisible, the toggle link or button will have class "hidden", otherwise class "showing" by default (see options classShowing and classHidden). The control is usually a child element of some "header" element (which may but does not have to contain a heading text for the section). And, depending on the heading element's class list, CSS rules can define different optical state representations for the section state, e.g. by changing an icon.
The default value of this toggle property is "#${id}_h". In this string, any occurrance of "${xyz}" will first be replaced by calling $(this).prop("xyz"). In this case, when showing or hiding the section with ID "mySection", the toggle selector will resolve to "#mySection_h". Then this selector will be used a jQuery search parameter to perform something like $("#mySection_h").addClass("showing").removeClass("hidden"). If you leave this setting on its default value, all you have to do is to make sure, that the ID if each toggle control equals the actual section's ID with appended suffix "_h". If you don't like this default convention, specify a different pattern string.
store string <optional>
 A pattern string similar to option toggle specifying a pattern for a jQuery selector string for finding a form element in which to store the current visibility state. The syntax is the same as for toggle. The default is undefined, which means that no such form control is used. If you are designing an HTML form with expandable and collapsable sections and you want the current state (whether a specific section is visible or hidden) to persist even when submitting the form, you may introduce hidden fields to your form, one per hideable section, and associate them with the sections by specifying this option. For example, if you set the ID of each of those hidden fields to the ID of the corresponding section with a suffix of "_state" appended, then set this option to: #${id}_state. Then for each section the state will be submitted the following way: The hidden field will be empty if the section is hidden, and the hidden field will be non-empty (value "X"), if the section is showing. Then your server application may evaluate this form data and take care of initialising each section's visibility in the response page it produces.
ariaExpandedSelector string ="#${id}_h a[href]:first, #${id}_h button:first" String pattern for a jQuery selector string, see option toggle. This default value is meant for the case that the toggle selector selects heading element containing a toggle control which is a button or a link. That's why this selector searches for the first link or button inside the section heading selected by "#${id}_h" (which is the default value for the toggle option, see above). If you change the toggle option, you should change this option accordingly. Or if your toggle selector does not select an element containing the actual toggle control (link or button), but the control itself, you should also change this option (to the same value as toggle in that case).
Meaning of this selector: For the element found by this selector (if any), the attribute "aria-expanded" will be set to true if the section is visible or false if the section is hidden. This is done for better accessibility: A screen reader will then not only read the toggle's text (link or button label), but will also announce the state (expanded or collapsed) of the section, and clicking that control will change this state.

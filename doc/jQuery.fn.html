(static) appendMoreLinks()(optionsopt) → {jqr}

jQuery plug-in for showing only the first / top part of a section and appending a link (like "more…") enabling users to show the rest of the section if they want to read it.

Take this HTML code for example:


<p class="appendMore">Lorem Ipsum …</p>
<div class="more">
    <p>Some more…</p>
    <p>Even more…</p>
</div>
With JavaScript disabled, a user will see all thee paragraphs. With JavaScript enabled you may execute a code like the following example in order to hide all ".more" blocks on load and append a "more…" link to each ".appendMore" block, wich, when clicked, will show the following ".more" block (and hide the link). 

$(function() {
    $(".more").hide();
    $(".appendMore").appendMoreLinks();
});
  • Of course, a web page may contains more than one pair of some node of class "appendMore" followed by a node of class "more".
  • The appendMoreLinks() function will simply append the links which will show the following ".more" node. It does not initially hide the ".more" sections, which you should do dynamically by javascript just like in the example above. This has the advantage over static style attributes in your HTML or a static CSS rule like (.more{display:none}) that this way, the whole content will be visible if JavaScript should be disabled by the user, which is also importent regarding accessibility.
  • Of course you may choose different class names. If the "more" blocks should use different classes than "more", don't forget to set the option moreContentSelector accordingly!.
  • The following ".more" node does not have to be descendant of the same parent node as the "appendMore" node, it may also be a descendant of one of the ".more"-links's parents, like for example: 
    <div>
    <p class="appendMore">Lorem Ipsum …</p>
</div>
<div class="more">…</div>
  • Nesting of "append-more-sections" is allowed. Example: 
    <p class="appendMore">Lorem Ipsum …</p>
<div class="more">
    <p> class="appendMore"gt;Some more…</p>
    <p> class="more"gt;Even more…</p>
</div>
  • Even simpler, a section may be of both classes for a multi-step revelation of following content, Example: 
    <p class="appendMore">Lorem Ipsum …</p>
<p class="more appendMore">Some more…</p>
<p class="more">Even more…</p>
  • When clicking the more link, the next "more" section will be shown with a vertical slide-down transition. But this only works if the "more" section is a block element (not an inline element).

(static) scrollDownIntoView(viewportopt, toleranceopt, durationopt) → {jqr}

jQuery plug-in function: Tries to scroll the selected section into view. The section is the first element of the matched elements (jQuery result set). If this sections height (minus twice the tolerance) is larger than the viewport's height, it will be aligned by its top (see viewportAlignTop), otherwise it will be aligned by its bottom (see viewportAlignBottom). If the current viewport is above the target element, this will result in scrolling down only as far as necessary to get the whole element into view (or to get its top into view, if it's too large for the viewport).

(static) showOrHideChildren(childrenSelect, optionsopt) → {jqr}

jQuery plug-in function: For each element of the jQuery resultset for which this function is called, search all decendents (children of the node as well as their children etc.) for matches of a child-selector (first argument) and either hide or show all those matching children. If no condition option is passed in the options argument, the following default condition will decide whether the children will be shown or hidden: If the selected heading element (see jQuery.fn.showOrHideSection() and jQuery.fn.showOrHideSection.DEFAULTS for details) has the class "showing" (more precisely the class defined in options.classShowing, which defaults to "showing", see jQuery.fn.showOrHideSection.DEFAULTS), then the children will be hidden (and the class "showing" will be removed from the heading and the class "hidden" (resp. options.classHidden) will be added).

For the default toggle condition to work, a heading element is obligatory. See heading option in Default Options! If the default value (selecting an element with the ID of the selected parent plus "_h" suffix) is not applicable in your code, don't forget to specify a suitable heading option in the arguments!

Just like jQuery.fn.showOrHideSections() this function changes the class list of the heading element (see previous paragraph) which enables you to specify some CSS for the state visualization, and it also supports specifying an aria-expanded attribute of a toggle element (see options) for better accessibility.

Supports almost every option that jQuery.fn.showOrHideSection() supports, see jQuery.fn.showOrHideSection.DEFAULTS for a list with their default values. Please not, though, that the "store" option is currently unsupported for this plug-in.

Three call signatures are supported:

  • $(parentSelector).showOrHideChildren(childrenSelector, options): This is the default signature and the only allowed one if no setup has been stored via jQuery.fn.setupShowOrHideChildren. The options argument is optional, the childrenSelector is required without a setup!
  • $(parentSelector).showOrHideChildren(options): This signature is only valid if a setup has been stored which at least defines a childrenSelector. Any options passed here will override options present in the setup, options not stated in this arguments will be taken from the setup (or from the DEFAULTS if not present in setup).
  • $(parentSelector).showOrHideChildren(): The parameterless call is also only allowed if a setup has been stored. That setup will then be used without overrides.

childrenSelect string Selector String for selecting those ancestor nodes of each selected node that should be shown or hidden. Usually a required argument, except if a setup has been stored which already defines such a childrenSelect selector. Then this argument on the actual showOrHideChildren() call is optional.
options object <optional>
 Options for this function. This is merged with the setup. If no options are given at all, the setup is used (see jQuery.fn.setupShowOrHideChildren). If no setup has been made, the DEFAULTS are used.

(static) viewportAlignBottom(viewportopt, toleranceopt, durationopt) → {jqr}

jQuery plug-in function: Tries vertical scroll in order to align the bottom of the first element in the set of matched elements (jQuery result set) with the bottom of the viewport. The alignment isn't always quite accurate, also an exact alignment with the bottom of the viewport often isn't desirable, a little margin between the viewport and the element may look better. That's why a tolerance is specified, defaulting to 30 px. This means, the plug-in will try to align the bottom of the matched element approximately 30px above the bottom of the viewport, the effective margin may be smaller.

Optional Dependency: jquery.scrollTo: If this plug-in is available, it will be used to scroll animatedly, otherwise the alignment will be instantaneous, without animation.

Usage pattern:

$("#mySection").viewportAlignBottom();
Parameters:

viewport jqr <optional>
 $(window) jQuery resultset, the first element in this set defines the viewport to align against. The default viewport is the whole window.
tolerance int <optional>
 30 approx. margin between bottom of the element and bottom of the viewport, see above
duration int <optional>
 200 duration of the scrolling animation. The default value of 200ms is defined in jQuery.fn.showOrHideSection.DEFAULTS, which may be overridden. (if the scrollTo plug-in is loaded).

(static) viewportAlignRight(viewportopt, toleranceopt, durationopt) → {jqr}

jQuery plug-in function: Tries horizontal scroll in order to align the right of the first element in the set of matched elements (jQuery result set) with the right of the viewport. (Horizontal version of jQuery.fn.viewportAlignBottom.)

Optional Dependency: jquery.scrollTo: If this plug-in is available, it will be used to scroll animatedly, otherwise the alignment will be instantaneous, without animation.

Usage pattern:

$("#mySection").viewportAlignRight();
Parameters:
Name Type Attributes Default Description
viewport jqr <optional>
 $(window) jQuery resultset, the first element in this set defines the viewport to align against. The default viewport is the whole window.
tolerance int <optional>
 30 approx. margin between right of the element and right of the viewport, see above
duration int <optional>
 200 duration of the scrolling animation. The default value of 200ms is defined in jQuery.fn.showOrHideSection.DEFAULTS, which may be overridden. (if the scrollTo plug-in is loaded).

(static) viewportAlignTop(viewportopt, toleranceopt, durationopt) → {jqr}

jQuery plug-in function: Tries vertical scroll in order to align the top of the first element in the set of matched elements (jQuery result set) with the top of the viewport. The alignment isn't always quite accurate, also an exact alignment with the top of the viewport often isn't desirable, a little margin between the upper window edge and the element may look better. That's why a tolerance is specified, defaulting to 20 px. This means, the plug-in will try to align the top of the matched element approximately 20px above the top of the viewport, the effective margin may be smaller.

Optional Dependency: jquery.scrollTo: If this plug-in is available, it will be used to scroll animatedly, otherwise the alignment will be instantaneous, without animation.

Usage pattern:

$("#mySection").viewportAlignTop();

Demo for appendMoreLinks

Demo 1: Default class names

  • Imagine your page contains longer text passages, of which you want to initially only present the beginning like in these list entries. If the user wants to read on, he should simply press a link to reveal the rest of the section:

    When the "more..." link gets pressed, the initially hidden remainder of the content gets shown and the link itself gets removed. Just like demonstrated here.

  • To achieve this, simply add the class appendMore to the element of the introducing text to which the "more..." link should be added, and…

    add the class more to the element with the remainder, which should be initially hidden and only revealed when the "more…" link is clicked.
    Note that the more part can also be an appendMore part, as is the case here: This part is of class more, hidden on page load and revealed by clicking the preceding "more…" link. But it's also of class appendMore, i.e. it ends with a "more…" link of its own which will reveal a third part.

    (This is the third part.)

  • Then you'll only need a few lines of JavaScript to enable the plug-in.

    $(function() {
    $(".more").hide(); //Initially hide alle "more" parts
    $(".appendMore").appendMoreLinks(); //And append the "more..." links to show them again to all "appendMore" parts.
});

  • If JavaScript is disabled (or jQuery or the plug-in are not found), then no content will be hidden (and no non-working "more…"-links will be visible). The user will then simply see all your content.

    You may try this by disabling JavaScript in your browser an reloading this page.

Demo 2: Custom class names and link labels

In Demo 1, default class names "more" and "appendMore" have been used and the generated links hat the default link label ("more…").

Now, suppose you want to…

  1. …use custom class names of your choice instead of "appendMore" and "more".

    In this demo, the first paragraph above has the class name "part1" (instead of "appendMore") and the "more" parts now use class name "part2".

    Then the script may look like this:

    $(".part2").hide();
$(".part1").appendMoreLinks({
    moreContentSelector: ".part2"
});

    Please note that it's not sufficient to only hide all ".part2" parts initially, but you also need to tell the appendMoreLinks() function that the inserted links now should look for a successor of class "part2" when clicked.

  2. and/or want to us define your own link label like this:
    That's what the moreLinkLabel option is for.
To summarize, the script code for Demo 2 looks like this: 
$(".part2").hide();
$(".part1").appendMoreLinks({
    moreContentSelector: ".part2",
    moreLinkLabel: "reveal more content…"
});
